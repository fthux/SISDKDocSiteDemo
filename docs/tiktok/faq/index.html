提示

以下问题主要出现在 iOS Tiktok v42.7.0 上，考虑到使用老版本的用户，使用新版本 iOS Tiktok 开发时，代码实现上也请注意兼容老版本。

iOS POST 请求传入的数据直接是字符串会被自动加上 data 封装 iOS Tiktok v42.7.0

调整请求入参形式，将原本复制 dataString 串改成 data 对象中的一个 key，后端取请求参数中的对应的 key 来处理。

参考代码
TTMinis.game.request({
  url: "https://xxxx.ccc.com/xxx/.json",
  method: "POST",
  data: {
    payload: dataString
  }
  }
})

iOS GET 求返回的数据被默认反序列化导致小游戏中再次反序列化报错。 iOS Tiktok v42.7.0

iOS 单端问题，data 被反序列，开发者代码中再次 JSON.parse 报错。

  • 方案一：在接口或者资源请求方法中，dataType 传入 'string'。
参考代码
TTMinis.game.request({
  url: "https://xxxx.ccc.com/xxx/.json",
  method: "GET",
  dataType: "string"
  }
})
  • 方案二：在接口返回阶段对 data 解析做兼容。
参考代码
TTMinis.game.request({
  url: "https://xxxx.ccc.com/xxx/xx.json",
  method: "GET",
  },
  success: (res) => {
    const data = safeJSONParse(res.data);
    // do next...
  }
})

function safeJSONParse(input) {
  // 请自行处理兼容逻辑
  ...
}
注意

如果使用 XMLHttpRequest 自行实现请求方法，即便设置 responseType 为 text，返回的 responseText 仍然是对象，请注意兼容。

iOS GET 请求框架带上额外通参导致游戏三方所带的参数因为 query 参数缺失出现请求异常。 iOS Tiktok v42.7.0

将 GET 请求方式调整成 POST 请求方式。

iOS 富文本由于读取 SDKVersion 版本不对展示异常，显示白底。 iOS Tiktok v42.7.0

修改构建路径下的 build/xxx/cocos-js/cc.xxxxx.js 中的 Number.parseInt(e.SDKVersion[0])<2 为 false。为了避免影响 Android 端，可修改为 e.platform=="android"&&Number.parseInt(e.SDKVersion[0])<2。

本地调试时，数据域好友排行榜显示正常，上传到后台测试，好友排行榜不显示。 iOS Tiktok Android Tiktok

使用调试工具 ttmg dev 开启本地调试后会在目录下生成 openDataContextCode.txt 文件，这个文件是排行榜必需的，需要在正式构建完成后，将 openDataContextCode.txt 复制到构建目录下。

iOS/Android 调用 FileSystemManager.unzip 报错导致游戏无法正常加载。 iOS Tiktok Android Tiktok

短期：开发者调整游戏内部需要解压消费文件的方式，改成不依赖 unzip。

长期：iOS/Android 支持上 FileSystemManager.unzip API。

本地调试时，iPhone 上无法正常播放激励。 iOS Tiktok

上传到后台进行测试。

本地调试时，iPhone 上无法存储本地数据。 iOS Tiktok

上传到后台进行测试。
