参数配置

配置一览​

需要配置的有：

  • TopOn 配置
  • AppsFlyer 配置
  • 内购盐值配置 无内购可忽略

SDK 初始化​

提示

SDK 初始化涉及到：SI 初始化、TopOn 配置、AppsFlyer 初始化。

SDK 初始化示例代码
private void Awake() {
    // 如果涉及到场景跳转，af 需要设置 DontDestroyOnLoad
    DontDestroyOnLoad(this);
}

private IEnumerator Start(){
    // SI 初始化
    yield return SkyIsland.SI.Init();

    // AppsFlyer 初始化
    AppsFlyer.initSDK("DEV_KEY", null);
    AppsFlyer.startSDK();
}

TopOn 配置​

配置 topon 的 appId 和 applicationKey，脚本路径 SkyIsland/Runtime/Scripts/Impl/HMS/Implementation.cs。

AppsFlyer 初始化​

AppsFlyer 初始化需要实现：

  • 1）DontDestroyOnLoad 涉及到场景跳转时必须，避免实例被销毁
  • 2）initSDK 方法传参：不是 iOS APP，所以 appid 为空即可。
AppsFlyer 初始化
private void Awake() {
    // 如果涉及到场景跳转，af 需要设置 DontDestroyOnLoad
    DontDestroyOnLoad(this);
}

AppsFlyer.initSDK("DEV_KEY", null);
AppsFlyer.startSDK();

AppsFlyer 调试​

提示

为了便于事件联调测试，可以显式设置 imei，af 后台添加该 imei 为测试设备即可正常联调：

设置 imei
AppsFlyer.setImeiData("YOUR_EMIT");

开启 debug 后，在 logcat 中过滤出 appsflyer 的日志。正常情况下，应该：日志显示对应请求响应成功，且 af 联调后台有 af_purchase af_ad_revenue launch 三个事件。

开启 debug
AppsFlyer.setIsDebug(true);
重要提示

正式上线前，请务必关闭设置 imei 代码和 debug 模式。
